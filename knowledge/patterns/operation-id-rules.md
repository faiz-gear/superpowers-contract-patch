# Operation ID Rules

`operationId` exists for machines as much as humans. Stable naming reduces client breakage and improves code generation.

Rules:

- every public endpoint has exactly one explicit `operationId`
- use action-oriented names tied to business intent
- keep naming stable unless there is a deliberate breaking change
- avoid collisions across versions or modules

Good:

- `createOrder`
- `getOrderById`
- `listOrders`

Bad:

- `controllerMethod`
- `index`
- autogenerated names that change with refactors
